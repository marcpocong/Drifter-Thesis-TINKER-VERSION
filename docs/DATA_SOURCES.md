# Data Sources and Provenance

This page lists external data and model/tool sources used by the workflow. It separates observation truth, transport validation, forcing, shoreline/oil support, and model/tool provenance.

Machine-readable registry: [config/data_sources.yaml](../config/data_sources.yaml)

Large external forcing datasets may not be fully stored in this repository. The repo preserves configs, manifests, inventories, source-taxonomy files, and derived stored outputs so a reviewer can trace what was used without rerunning science.

## What is not claimed

- Drifters support recipe selection, not direct oil-footprint truth.
- Public spill extents support B1/DWH spatial scoring only after common-grid processing.
- Forcing data are model inputs, not validation targets.
- PyGNOME is comparator-only.
- DWH is external transfer validation only.
- Mindoro oil-type and shoreline outputs are support/context only.
- `prototype_2016` is legacy/archive only.
- The repo does not make operational-system, grid-exact, or all-case accuracy claims for March 13-14; it does treat March 13 and March 14 as independent NOAA-published day-specific public-observation products.

## How to verify from the repo

1. Read the machine-readable registry at [config/data_sources.yaml](../config/data_sources.yaml).
2. Check Mindoro observation provenance in `output/CASE_MINDORO_RETRO_2023/phase3b_extended_public/extended_public_obs_inventory.csv`, `output/CASE_MINDORO_RETRO_2023/phase3b_multidate_public/phase3b_public_source_taxonomy.csv`, and the March 13-14 run manifest.
3. Check DWH observation provenance in `output/CASE_DWH_RETRO_2010_72H/phase3c_external_case_setup/`.
4. Check Phase 1 drifter and forcing provenance in `config/phase1_mindoro_focus_pre_spill_2016_2023.yaml`, `config/recipes.yaml`, and `output/phase1_mindoro_focus_pre_spill_2016_2023/phase1_production_manifest.json`.
5. Check stored forcing windows in the Mindoro and DWH forcing manifests; do not infer exact API endpoints that are not written there.
6. Check shoreline support in `data_processed/grids/shoreline_mask_manifest.json`.
7. Check oil-type support in `config/oil.yaml`, `config/phase4_oil_scenarios.csv`, and `output/phase4/CASE_MINDORO_RETRO_2023/`.
8. Check model/tool provenance in `pyproject.toml`, `Dockerfile.gnome`, and the final stored output manifests.
